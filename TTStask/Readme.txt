Структура приложения:
1) Клиентское приложение WPF
2) Сервер ASP.NET

Функционал клиентского приложения:
1) Кнопка запуск/стоп для начала отслеживания мыши и записи в БД
2) Таблица событий: (дата/время, событие, координаты)
3) Фильтрация значений в таблице по дате/времени и событию
4) Аутентификация админ/пользователь
5) Счётчик записей событий
6) Всплывающее окно с предложением отправить на указанный адрес общее количество записей

Функционал сервера:
1) Авторизации 
2) Логгирование событий(подключение к БД, аутентификация, запуск/остановка записи)
3) Отправка сообщений на указанный адрес

Функционал БД:
1) Записи в таблице выглядят следующим образом: дата/время,название события(сдвиг на 10px, клик правой, левой или средней),координаты
2) SQL скрипт создания БД
3) Используются  SQLite и Entity Framework
